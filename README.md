# MouseGestureApp

Windows全体で使えるマウスジェスチャー常駐アプリのサンプルです。マウスの右ボタンを
押したままドラッグすると軌跡が表示され、離した瞬間にジェスチャーを判定して
対応する操作(戻る/進む/最大化/最小化/閉じる)を実行します。

## 動作環境

- Windows 10 / 11
- Visual Studio 2022 (「.NET デスクトップ開発」ワークロードが必要)
- .NET 8 SDK

## 開き方

1. `MouseGestureApp.sln` をVisual Studioでダブルクリックして開く
2. スタートアップに `MouseGestureApp` が選択されていることを確認
3. F5 (デバッグ実行) または Ctrl+F5 (デバッグなし実行)

ウィンドウは表示されず、タスクトレイにアイコンが常駐します。終了する場合は
トレイアイコンを右クリックして「終了」を選んでください。

## 使い方(標準のジェスチャー)

右ボタンを押したまま、以下のようにマウスを動かして離します。

| ジェスチャー | 動作 |
|---|---|
| ↑ (上に動かす) | アクティブウィンドウを最大化 |
| ↓ (下に動かす) | アクティブウィンドウを最小化 |
| ← (左に動かす) | 戻る (Alt+Left) |
| → (右に動かす) | 進む (Alt+Right) |
| ↓→ (下、続けて右) | アクティブウィンドウを閉じる |

一定距離(既定12px)未満しか動かさずにボタンを離した場合は、通常の右クリックと
して扱われ、いつも通りコンテキストメニューが開きます。

## ジェスチャーを追加・編集する(設定画面)

コードを書かずに、タスクトレイアイコンを右クリック→「設定...」(またはトレイ
アイコンをダブルクリック)から追加・編集・削除できます。

1. 「追加」を押す
2. 「なぞって記録」を押してから、実際に右ボタンを押しながらジェスチャーを描く
   → コード欄(U/D/L/Rの組み合わせ)に自動入力される
3. 名前と動作を選ぶ。「キーボードショートカットを送信」を選んだ場合は、
   `Ctrl+W` や `Win+D` `Alt+F4` のような形式でキーを入力する
4. OK → 一覧画面で「保存」

設定は `%AppData%\MouseGestureApp\gestures.json` に保存され、アプリを
再ビルドしなくても次回起動時に引き継がれます。同じコードを別の動作に
上書き登録することもできます(確認ダイアログが出ます)。

### 送信できるキーの例

`Ctrl` `Alt` `Shift` `Win` を `+` でつなぎ、最後に本体のキーを1つ指定します。
本体のキーは `A`〜`Z` `0`〜`9` `F1`〜`F24` `Escape`(`Esc`) `Delete`(`Del`)
`Enter` `Tab` `Space` `Left` `Right` `Up` `Down` など、一般的なキー名が
そのまま使えます(大文字小文字は区別しません)。

### コードから拡張したい場合

設定画面を使わず、既定値そのものを変えたい場合は `GestureConfigStore.CreateDefaults()`
を編集してください。実行の中身を増やしたい場合(最大化/最小化/戻る/進む/閉じる/
キー送信、以外の全く新しい動作の種類を足したい場合)は `GestureBinding.cs` の
`GestureActionType` と `GestureActions.Execute` の `switch` に追加します。

## 実装のポイント

- `MouseHookService.cs`: `SetWindowsHookEx(WH_MOUSE_LL, ...)` によるグローバル
  マウスフック。右ボタンの押下は常に一旦抑制し、動きが小さいまま離された場合だけ
  `mouse_event` で本物の右クリックとして再生することで、通常の右クリック
  (コンテキストメニュー)を壊さないようにしています。
- `GestureOverlayWindow`: 全モニターを覆う透明・クリックスルーのWPFウィンドウで、
  ジェスチャーの軌跡を`Polyline`で描画します。
- タスクトレイアイコンは `System.Windows.Forms.NotifyIcon` を利用しているため、
  プロジェクトは WPF と Windows Forms の両方を有効化しています
  (`UseWPF` / `UseWindowsForms` を `csproj` で `true` に設定)。

## 既知の制限

- 管理者権限で動作しているウィンドウ(タスクマネージャーなど)に対しては、
  本アプリも管理者として実行しないとフックが効かないことがあります。
- 右ボタンのドラッグを別の用途に使うアプリ(一部の地図/CADソフトなど)とは
  操作がぶつかる可能性があります。その場合は `MouseHookService.cs` の
  `WM_RBUTTONDOWN` / `WM_RBUTTONUP` を別のトリガー(中ボタンなど)に
  変更してください。
- マルチモニターでDPI設定が異なる環境では、軌跡表示の位置がわずかにずれる
  場合があります。
